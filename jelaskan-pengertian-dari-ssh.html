3. Jelaskan bagaimana proses ssh menyampaikan data dari ssh client sampai ssh server?


Proses ssh menyampaikan data dari ssh client sampai ssh server adalah ketika Client dan server setuju untuk menggunakan sesi SSH tertentu, clien meminta public key kemudian client dan server menyetujui alogaritma dan membentuk suatu session key. Kemudian server mendecrypt session dan mengirimkannya kembali ke client untuk verifikasi. Terakhir, pemakai mengotentikasi dirinya ke server didalam aliran data terenskripsi dalan session key dan clien selanjutnya bekerja secara interaktif pada server mentransfer file ke atau dari server.  

Pembahasan  

SSH adalah protokol aman yang digunakan sebagai sarana utama untuk menghubungkan ke server Linux dari jarak jauh. Menyediakan antarmuka berbasis teks dengan pemijahan shell jarak jauh. Setelah tersambung, semua perintah yang di ketik di terminal lokal dikirim ke server jarak jauh dan dijalankan di sana.

Cara paling umum menghubungkan ke server Linux remote adalah melalui SSH. SSH adalah singkatan dari Secure Shell dan menyediakan cara yang aman dan aman untuk mengeksekusi perintah, membuat perubahan, dan mengonfigurasi layanan dari jarak jauh. Ketika terhubung melalui SSH, maka akan masuk menggunakan akun yang ada di server jauh.

Cara kerja SSH

Ketika   terhubung melalui SSH, maka akan jatuh ke sesi shell, yang merupakan antarmuka berbasis teks di mana  dapat berinteraksi dengan server . Selama sesi SSH  , setiap perintah yang di ketikkan ke terminal lokal   dikirim melalui terowongan SSH terenkripsi dan dijalankan di server.

Koneksi SSH diimplementasikan menggunakan model client-server. Ini berarti bahwa untuk koneksi SSH yang akan ditetapkan, mesin remote harus menjalankan perangkat lunak yang disebut daemon SSH. Perangkat lunak ini mendengarkan koneksi pada port jaringan tertentu, mengotentikasi permintaan koneksi, dan memunculkan lingkungan yang sesuai jika pengguna memberikan kredensial yang benar.

Komputer pengguna harus memiliki SSH klien. Ini adalah perangkat lunak yang mengetahui cara berkomunikasi menggunakan protokol SSH dan dapat diberikan informasi tentang host jarak jauh untuk terhubung, nama pengguna yang digunakan, dan kredensial yang harus diloloskan untuk mengotentikasi. Klien juga dapat menentukan detail tertentu tentang jenis koneksi yang ingin mereka buat.

Klien umumnya mengotentikasi baik menggunakan sandi (kurang aman dan tidak disarankan) atau kunci SSH, yang sangat aman.

Login sandi dienkripsi dan mudah dimengerti bagi pengguna baru. Namun, bot otomatis dan pengguna jahat sering kali berulang kali mencoba mengautentikasi ke akun yang memungkinkan proses masuk berbasis kata sandi, yang dapat mengarah pada kompromi keamanan. Untuk alasan ini, kami menyarankan untuk selalu menyiapkan autentikasi berbasis kunci SSH untuk sebagian besar konfigurasi

Kunci SSH adalah satu set kunci kriptografi yang cocok yang dapat digunakan untuk otentikasi. Setiap set berisi kunci publik dan pribadi. Kunci publik dapat dibagikan secara bebas tanpa kepedulian, sementara kunci privat harus dijaga dengan waspada dan tidak pernah diperlihatkan kepada siapa pun.

Untuk mengotentikasi menggunakan kunci SSH, pengguna harus memiliki pasangan kunci SSH di komputer lokal mereka.

Menghasilkan Pasangan Kunci SSH

Menghasilkan pasangan kunci publik dan pribadi SSH baru di komputer lokal   adalah langkah pertama menuju autentikasi dengan server jauh tanpa kata sandi. Kecuali ada alasan kuat untuk tidak melakukannya,   harus selalu mengotentikasi menggunakan kunci SSH.

Sejumlah algoritma kriptografi dapat digunakan untuk menghasilkan kunci SSH, termasuk RSA, DSA, dan ECDSA. Kunci RSA umumnya lebih disukai dan merupakan jenis kunci default.

12. jelaskan hubungan antara ftp secure dengan command ssh​


FTP Secure (FTPS) adalah protokol transfer file yang menggunakan enkripsi untuk melindungi keamanan transfer data. FTPS menggunakan keamanan SSL/TLS untuk melindungi data saat transit antara client dan server. Ini memastikan bahwa data yang ditransfer tidak dapat dicegat atau dimodifikasi oleh pihak yang tidak berwenang.

Command SSH (Secure Shell) adalah protokol jaringan yang dirancang untuk mengamankan komunikasi antara dua perangkat, seperti komputer client dan server, melalui jaringan yang tidak aman. Melalui SSH, pengguna dapat mengakses dan mengendalikan perangkat dari jarak jauh dengan aman. SSH menggunakan enkripsi simetris dan asimetris untuk mengamankan koneksi dan melindungi data dari serangan pihak ketiga.

Hubungan antara FTP Secure dan command SSH adalah bahwa keduanya dirancang untuk meningkatkan keamanan dalam proses transfer data atau akses ke perangkat yang aman.

Meskipun keduanya berfokus pada keamanan, FTPS dan SSH berbeda dalam cara mereka bekerja. FTPS beroperasi dalam lapisan aplikasi dan memberikan enkripsi end-to-end untuk transfer file melalui protokol FTP. Sementara itu, SSH beroperasi lebih rendah dalam lapisan transportasi dan memberikan enkripsi end-to-end untuk mengamankan koneksi jaringan, serta memberikan akses baris perintah ke perangkat yang aman.

Dalam beberapa kasus, SSH dapat digunakan untuk mengamankan akses ke server FTPS. Dalam skenario ini, pengguna akan terlebih dahulu menggunakan SSH untuk terhubung ke server secara aman, dan kemudian mereka dapat menggunakan protokol FTPS untuk mentransfer file dengan aman melalui koneksi yang dienkripsi oleh SSH.

Dengan menggunakan kedua teknologi ini, pengguna dapat menjaga keamanan data selama transfer file dan akses jarak jauh mereka, memastikan bahwa informasi sensitif tetap terlindungi dari potensi serangan atau pengintaian.

18. Jelaskan langkah-langkah tata kerja dalam melakukan koneksi pada ssh server


Jawaban:

berikut adalah beberapa tahapan-tahapannya:

1 . SSH client melakukan koneksi ke SSH server menggunakan port default 22.

2. Jika SSH client dan SSH server menggunakan versi yang sama, koneksi akan sukses.

3. Selanjutnya, sSH client meminta public key dan host key dari SSH server dan melakukan verifikasi.

4. Jika kedua kunci yang digunakan sama, SSH client akan membentuk session key dan mengenkripsi kunci tersebut melalui public key milik SSH server.

5. Kemudian, SSH server akan mengenkripsi balik public key milik SSH client untuk verifikasi.

6. User dari SSH client lalu harus melakukan otentikasi diri dengan cara memasukan username dan password yang sudah ada.

Setelah verifikasi pengguna berhasil, koneksi sudah terbentuk dengan aman. Sekarang Anda sudah bisa menggunakan SSH client untuk melakukan koneksi jarak jauh ke server Anda!

31. Kenapa VPN lebih aman daripada SSH


Konfigurasi pada client, SSH Tunneling tidak menawarkan semua manfaat dari VPN. Berbeda dengan VPN, Anda harus mengkonfigurasi setiap aplikasi untuk menggunakan proxy SSH Tunneling tersebut. Dengan VPN, semua lalu lintas di komputer clien akan secara otomatis masuk kedalam “Private Network”, yang mana Anda tidak memiliki jaminan ini dengan sebuah SSH Tunnerling. Dengan VPN, sistem operasi Anda akan bersikap seolah-olah Anda berada di jaringan remote. Ini jauh lebih sulit dengan sebuah terowongan SSH.

34. Apakah pengertian dari database, server, proxy, ssh, dan ip :)


database, menurut saya adalah sekumpulan data yang telah tersimpan dan diatur, sehingga bisa dicari atau diambil debngan efisien ...
server, sebuah sistem komputer yang menyediakan jenis layanan tertentu dalam sebuah jaringan komputer ( kurang paham )
proxy, suatu mekanisme dan service serta jembatan anda ke facebook.com sekaligus memberikan konten yang sudah/bisa di cache/save di direktory proxy server untuk saat nanti jika anda mengakses ke facebook.com lagi tanpa harus mengambil langsung dari server facebook. 
ssh, (penjelasanya susah, tapi saya sering gunain ssh untuk gratisan)
ip, nama komputer anda dalam nilai angka di jaringan, komputer mengenali komputer lainnya dengan angka misal 192.168.1.100 bukannya nama dalam huruf seperti hostname atau domain seperti facebook.com. 

database = kumpulan informasi yang disimpan di dalam komputer secara sistematik untuk memperoleh informasi dari basis data tersebut.
server = sebuah sistem komputer yang menyediakan jenis layanan tertentu dalam sebuah jaringan komputer.
proxy = suatu server yang menyediakan layanan untuk meneruskan setiap permintaan kita kepada server lain di internet.
ssh = (Secure Shell) protokol jaringan yang memungkinkan pertukaran data melalui saluran aman antara dua perangkat jaringan.
ip = (internet protocol) suatu cara untuk mengetahui asal atau alamat suatu komputer berupa sistem penomoran masing-masing komputer yang bersifat unik atau tidak sama. Sistem penomoran itu sendiri terdiri dari empat bagian yang dipisahkan oleh titik contoh : 202.155.245.2
